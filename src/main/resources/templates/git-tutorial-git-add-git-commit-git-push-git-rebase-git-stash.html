Git Basic Commands

Steps to resolve merge conflicts in git:

Step1) Fetch the target branch. E.g. master or dev 

git fetch origin master

Step2) Go to your feature branch 

git checkout -b 'featurebranch' 'origin/featurebranch'

Step3) Do rebase 

git rebase origin/master

After running above command conflict file will be displayed on the command line. Also, the conflict file will be highlighted in the IDE

Step4) Remove the code which is not required along with the comments generated by the merge conflict.

Step5) Add the changes 

git add .

Step6) Commit the changes 

git commit -m "fixed merge conflicts"

Step7) Now run rebase continue command 

git rebase --continue

Step8) Force push the changes to feature branch 

git push --force origin featurebranch

Now if you check the merge request in github you can see there will not be any merge conflicts

git stash:

Step1) To checkout other branch with uncommitted files in current branch 

git stash
git checkout otherBranchName

Step2) To get back uncommitted files 

git checkout currentBranchName
